# DjangoQL 搜索语法

## 搜索条件

搜索条件是构建搜索查询的基本单元。它始终由 3 个元素组成：`field`（字段）、`comparison operator`（比较运算符）和 `value`（值），且必须严格按照从左到右的顺序排列。

以下是一个示例——查找名字为"John"的用户。在此示例中，`first_name` 是 `field`，`=` 是 `comparison operator`，`"John"` 是 `value`：

```
first_name = "John"
```

另一个示例，查找 2017 年或之后注册的用户：

```
date_joined >= "2017-01-01"
```

再看一个示例，查找超级用户：

```
is_superuser = True
```

还有一个示例——查找所有名字在指定列表中的用户：

```
first_name in ("John", "Jack", "Jason")
```

## 多个搜索条件

您可以使用逻辑运算符将多个搜索条件组合在一起：`and`（两个条件必须同时为真）和 `or`（至少一个条件为真，不限哪一个）。注意——逻辑运算符必须小写：`and` 和 `or` 是正确的，而 `AND` 或 `OR` 是错误的，会导致报错。

示例：查找名字为"John" `and` 2017 年或之后注册的用户。请注意，这里有 2 个搜索条件，用 `and` 连接：

```
first_name = "John" and date_joined >= "2017-01-01"
```

再看一个示例，查找超级用户 `or` 标记为"Staff"的用户：

```
is_superuser = True or is_staff = True
```

逻辑运算符功能强大，可用于构建复杂的搜索查询。在构建复杂查询时，有一个重要提示需要记住：如果查询中同时包含 `and` 和 `or` 运算符，强烈建议使用括号来明确运算符的优先级。以下示例说明了这一点的重要性。假设您想查找超级用户 `or` 标记为 Staff 的用户，`and` 这些用户都是 2017 年或之后注册的。您可能会这样写：

```
is_superuser = True or is_staff = True and date_joined > "2017-01-01"
```

上述查询的问题在于它不会按预期执行，因为 `and` 运算符会被优先计算。实际上，它查找的是超级用户（无论何时注册）`or` 同时满足 Staff `and` 2017 年之后注册的用户。可以用括号来解决这个问题，只需将必须优先计算的搜索条件用括号括起来，如下所示：

```
(is_superuser = True or is_staff = True) and date_joined > "2017-01-01"
```

仅当查询同时混合使用 `and` 和 `or` 运算符时，才建议使用括号。如果查询中只包含同一种逻辑运算符（仅 `and` 或仅 `or`），则可以安全地省略括号，查询将按预期执行。

## 字段

在搜索查询中，您应该按照 Django 模型的 Python 代码中的定义来引用当前模型的字段。搜索查询输入框具有自动补全功能，会自动弹出并显示所有可用选项。如果您不确定字段名称，可以从显示的选项中选择（示例）：

![DjangoQL 自动补全示例](COMPLETION_EXAMPLE_IMG)

在大多数情况下，Django 模型内部字段的名称与您在 Django 管理界面中看到的名称类似，只是全部小写且用 `_` 代替空格。例如，在标准用户管理界面中，内部字段 `first_name` 显示为"First name"，`email` 字段显示为"Email address"，以此类推。但也可能存在例外情况，即开发者定义了与内部名称差异很大的自定义显示名称。在这种情况下，可以请开发者覆盖此帮助模板，并在此处提供"内部名称 -> 显示名称"的字段映射。

请注意，Django 管理界面中显示的某些字段可能无法被搜索。这包括计算字段，即那些不作为普通值存储在数据库中，而是在代码中由其他值计算得出的字段。

## 关联模型

DjangoQL 也支持按关联模型进行搜索（它会在底层自动将关联关系转换为 SQL 连接）。使用 `.` 点号分隔符来指定关联模型及其字段。例如：

```
groups.name in ("Marketing", "Support")
```

注意上面示例中的 `.` 吗？它表示 `groups` 是一个关联模型，`name` 是该模型的一个字段。与往常一样，DjangoQL 自动补全功能会为所有可用的关联模型及其字段提供建议。对于复杂的数据结构，您可以使用多级关联，即先指定一个关联模型，再指定它的关联模型，以此类推。

在大多数情况下，包含关联模型的搜索条件必须指定该模型的具体字段，而不能只写关联模型本身。例如，`groups in ("Marketing", "Support")` 无法工作，因为 `groups` 是一个模型而非字段。模型可以有很多字段，服务器无法知道您希望与哪个字段进行比较。不过有一个值得注意的例外——当您希望查找与任意关联模型关联（或未关联）的记录时，应将关联模型与特殊值 `None` 进行比较，如下所示：

```
groups = None
```

上述示例将查找不属于任何分组的用户。如果您想查找至少属于一个分组的所有用户，请使用 `!= None`：

```
groups != None
```

## 比较运算符

| 运算符 | 含义 | 示例 |
| --- | --- | --- |
| `=` | 等于 | `first_name = "John"` |
| `!=` | 不等于 | `id != 42` |
| `~` | 包含子字符串 | `email ~ "@gmail.com"` |
| `!~` | 不包含子字符串 | `username !~ "test"` |
| `startswith` | 以子字符串开头 | `last_name startswith "do"` |
| `not startswith` | 不以子字符串开头 | `last_name not startswith "do"` |
| `endswith` | 以子字符串结尾 | `last_name endswith "oe"` |
| `not endswith` | 不以子字符串结尾 | `last_name not endswith "oe"` |
| `>` | 大于 | `date_joined > "2017-02-28"` |
| `>=` | 大于或等于 | `id >= 9000` |
| `<` | 小于 | `id < 9000` |
| `<=` | 小于或等于 | `last_login <= "2017-02-28 14:53"` |
| `in` | 值在列表中 | `first_name in ("John", "Jack", "Jason")` |
| `not in` | 值不在列表中 | `id not in (42, 9000)` |

注意：

1. `~` 和 `!~` 运算符只能用于 string 和 date/datetime 字段。date/datetime 字段将被当作字符串字段处理（例如，`payment_date ~ "2020-12-01"`）
2. `startswith`、`not startswith`、`endswith` 和 `not endswith` 只能用于 string 字段；
3. `True`、`False` 和 `None` 值只能与 `=` 和 `!=` 组合使用；
4. `in` 和 `not in` 运算符必须小写。`IN` 或 `NOT IN` 是错误的，会导致报错。

## 值

| 类型 | 示例 | 说明 |
| --- | --- | --- |
| string | `"this is a string"` `'another string'` | 字符串可以用双引号（如 `"this"`）或单引号（如 `'this'`）括起来。如果字符串中包含与外层引号相同类型的引号，应使用反斜杠进行转义，例如 `"this is a string with \"quoted\" text"` 或 `'this is a string with \'quoted\' text'`。 |
| int | `42`, `0`, `-9000` | 整数即可选带一元负号的数字。输入较大数字时请勿使用千位分隔符，DjangoQL 无法识别它们。 |
| float | `3.14`, `-0.5`, `5.972e24` | 浮点数看起来像带有可选小数部分（用点号分隔）的整数。也可以使用 `e` 表示法来指定 10 的幂次。例如，`5.972e24` 表示 5.972 × 10^24。 |
| bool | `True`, `False` | 布尔值是一种特殊类型，只接受两个值：`True` 或 `False`。这些值区分大小写，必须严格按照此格式书写——首字母大写，其余小写，不加引号。 |
| date | `"2017-02-28"` | 日期以 `"YYYY-MM-DD"` 格式的字符串表示。 |
| datetime | `"2017-02-28 14:53"` `"2017-02-28 14:53:07"` | 日期和时间可以用 `"YYYY-MM-DD HH:MM"` 格式的字符串表示，也可以选择带秒数的 `"YYYY-MM-DD HH:MM:SS"` 格式（24 小时制）。请注意，日期和时间的比较以服务器时区为准，通常为 UTC。 |
| null | `None` | 这是一个表示"无任何值"的特殊值：`None`。必须严格按照此格式书写——首字母大写，其余小写，不加引号。当数据库中某个字段可为空（即 SQL 中可以为 NULL）时使用它，可用于查找无值的记录（`some_field = None`）或有值的记录（`some_field != None`）。 |
